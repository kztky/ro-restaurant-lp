# 仮想飲食店のLP制作 — 炉 RO

---

## ① ターゲット

**メインターゲット**
30〜55歳、経済的余裕のある都市部在住の男女。国内外問わず、非日常体験に価値を見出す層。

**ペルソナ**
- 東京・大阪在住、年収800万円以上。旅行先で「知る人ぞ知る店」を好む
- SNSで映える体験よりも、語れる体験・選んだ理由を説明できる体験を求めている
- インバウンド層：東アジア（中国・台湾・韓国）を中心とした富裕層旅行者

**地理的文脈**
青森という「知名度はあるが観光地として洗練されていない」土地の希少性を逆手に取る。わざわざ来る価値のある体験として設計。

## ② この人に刺さる要素の推定

| 要素 | 狙い |
|---|---|
| 炭火・発酵というキーワード | 「手間をかけている」ではなく「哲学がある」と伝わる |
| 余白と静寂のビジュアル | 情報過多な日常への対比。「ここでは何も考えなくていい」という安心感 |
| 完全予約制・8席 | 希少性と特別感。大衆に開かれていないことがむしろ価値になる層 |

| 青森・津軽という産地の明示 | 「どこかの食材」ではなく「ここにしかない食材」というストーリーの具体性 |

## ③ コンセプトメイキング

**コアコンセプト：「火と菌がつくる、時間」**

津軽の短い夏が食材に旨みを凝縮させ、炭火と発酵がそれをゆっくりと解き放つ。料理を「作るもの」ではなく「時間の結晶」として再定義した。炭火の遠赤外線と発酵は、どちらも「急がず、ゆっくりと作用する力」であり、せわしない日常との対比を生む。

青森・津軽という土地の必然性をコンセプトの核に置いた。豪雪地帯の長い冬と短く強烈な夏が、りんごの糖度を上げ、発酵文化を根付かせ、炭火文化を必要とさせた。「ここにしかない素材だから、ここにしかない夜がある」という論理構造がサイト全体を貫いている。

**Heroコピー**
> 短い夏に凝縮された、津軽の旨みがある。
> 炭火がゆっくりと、雪解けのように解く。
> 素材が語る言葉に、ただ耳を傾ける夜。

1行目で場所の必然性を、2行目で調理哲学を「雪解け」という津軽固有のイメージで表現し、3行目で体験への誘いとして締める三段構成。

**店舗設計とブランドの整合**

カウンター8席・完全予約制・個室なしという制約をすべてブランドの一部として設計した。個室がないことは「炉を囲む一体感を大切にするため」と明示し、デメリットではなくコンセプトの表明として機能させた。価格は6品¥30,000（税込）の固定価格とし、明朗会計を担保しつつドリンクペアリング等でアップセルの余地を残した。

**視覚言語の設計**
- カラーパレット：インク（#1a1510）・燠火（#c4783a）・金（#b8953f）の3色に限定
- フォント：しっぽり明朝（和文）× Cormorant Garamond（欧文）— ともに「余白を持つ」書体
- インタラクション：カスタムカーソル、スクロール連動調光など、操作そのものが体験になるよう設計

**店名「炉 RO」**
漢字一文字で完結する命名。火を囲む原始的な行為への回帰と、英語圏への音の通りやすさを両立。

## ④ 制作の流れと判断のプロセス

### 設計フェーズ
「何を見せるか」より先に「何を感じさせるか」を定義した。Hero・コンセプト・メニュー・空間・予約という感情の動線を先に設計し、その後に各要素を実装した。

### ナビゲーション設計
デスクトップ用ナビリンクとモバイル用ハンバーガーを分けて実装していたが、lang-switcherとの干渉が発生。デスクトップでもハンバーガーに統一し、lang-switcherを常時中央表示する構成に変更した。インバウンド向けサイトとして言語切り替えの即アクセス性を優先した判断。

### メニュー表示の設計
グリッドは写真＋タイトルのみ、詳細はホバーで右パネルに展開する構造を採用。「見せる前に期待させる」設計がターゲット層の購買行動に近いと判断した。モバイルではアコーディオン形式に切り替え、デバイスごとに最適なUXを維持した。

### その他の技術的判断
- タイポグラフィ：ラベルは`font-size: 12px / letter-spacing: 0.5em`、h2は`clamp(28px, 3.5vw, 44px)`に全セクション統一
- 予約ブロック：`align-items: stretch`で両カラムの高さを揃え、視覚的安定感を確保
- セクション調光：IntersectionObserverでスクロール連動の明暗制御を実装

## ⑤ 仮想ペルソナ評価による仮説検証

初稿完成後、3種の仮想評価者ペルソナを設定し、「ポートフォリオ閲覧者」と「エンドユーザー」の両視点からサイトを評価した。ペルソナはAIが演じ、属性設計・評価軸・採否の判断は人間が行った。

**評価者**：Web制作AD（国内・女性）、香港在住富裕層旅行者（海外・男性）、東京在住ITコンサル部長（国内・男性・年収1,100万）

**評価から変えたもの**：モバイルのphoto-strip高さ、lang-switcherとハンバーガーの干渉、メニュープレースホルダーの視認性、tax表記の多言語対応、Heroの予約リンク配置、ナビの言葉（概念→コンセプト、献立→メニュー）、Heroコピー、コース価格の明示、個室なしの明示

**評価から変えなかったもの（と理由）**

- **固定予約バナーの追加** → 高単価帯のブランドに合わない。「そこで離脱する人はターゲットではない」という設計思想と整合する。予約意欲はボタンの数ではなくサイト全体の体験で作るものと判断した
- **コンセプト文の短縮** → 流し読みしない層がターゲットであるため、長文は弱点ではなく選別として機能する
- **シェフの顔出し** → 人物を前に出すとブランドの匿名性・静寂感が失われる。炉 ROのトーンと相反する

## ⑥ 制作情報

| 項目 | 内容 |
|---|---|
| 制作時間 | 約4〜5時間（構想・設計・実装・修正）＋評価・改善セッション |
| 使用ツール | Claude（Anthropic）によるAI協働制作 |
| 実装 | HTML / CSS / JavaScript（バニラ、フレームワーク不使用） |
| フォント | Google Fonts（しっぽり明朝・Cormorant Garamond・Noto Sans多言語） |
| 対応言語 | 日本語・英語・中国語簡体字・繁体字・韓国語 |
| 対応デバイス | デスクトップ・モバイル（レスポンシブ） |

**AIとの協働について**

本制作はClaude（Anthropic）との対話形式で進めた。コードの生成・修正・ペルソナの演技はAIが担当し、設計の判断・ターゲット定義・UXの方向性・採否の意思決定は人間が行った。

「AIが作った」ではなく「AIを使って作った」という区別は重要で、品質の担保と意思決定は一貫して人間側にある。この制作スタイル自体が、現在のWeb制作の現場で求められる能力のひとつだと考えている。
